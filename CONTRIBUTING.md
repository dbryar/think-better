# Contributing

This project accepts focused pull requests that improve the field guide, correct mistakes, or make the publishing pipeline easier to inspect.

Use GitHub Issues for factual corrections, accessibility problems, design issues, and feature requests:

https://github.com/dbryar/think-better/issues

Before opening a pull request:

- Run `bun run ci`.
- Keep factual claims checkable and sourced.
- Teach methods rather than loyalty.
- Avoid party-political endorsements in core material.
- Check the governance review checklist.

Content lives in `content/` and governance material lives in `governance/`. Use the existing frontmatter fields so the site, validation script, and release process can read the material consistently.
